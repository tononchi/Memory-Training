# Memory-Training

読みの力を育てる Web アプリ集です。**学年帯ごとに有効な手法が異なる**という文献調査（[docs/reading-by-grade-band.md](docs/reading-by-grade-band.md)）に基づき、2つのアプリを用意しています。

| アプリ | 対象 | 手法（エビデンスに基づく） |
|---|---|---|
| **よむよむスピードトレーニング**（`index.html`） | 小学1〜6年＋大人 | 反復読み（流暢性／自動化）＋お手本＋理解確認。反復読みは小6まで有効 |
| **速読・読解トレーニング**（[`secondary/index.html`](secondary/index.html)） | 中学生・高校生 | 目的に応じた読速の使い分け（スキミング／スキャニング／精読）＋読解方略＋理解の担保 |

> なぜ分けるか：メタアナリシスでは、流暢性ドリルの効果は学年が上がるほど縮小し、音読速度は小6前後で頭打ちになります。中学生以降は「速く読む」こと自体より、読解方略・語彙・目的に応じた読速調整が効きます。いわゆる高速提示型の“速読”（RSVP）は速度こそ上がりますが詳細理解は向上しません。詳細は[docs/reading-by-grade-band.md](docs/reading-by-grade-band.md)。

ローカル実行でも、静的ホスティング（GitHub Pages / Netlify / Vercel など）でも動作します。

## 使い方（ローカル）

### 1) ファイルを直接開く
`index.html` をブラウザで開くだけで実行できます。

### 2) 簡易サーバーで実行（推奨）

```bash
cd Memory-Training
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開きます。

## よむよむスピードトレーニング（小学生むけ）の機能

文献調査（[docs/speed-reading-research.md](docs/speed-reading-research.md)）の知見に沿った設計です。

- **反復読み（Repeated Reading）**: 同じ短文を2〜4回くりかえし読み、流暢性を育てる（中核手法）
- **お手本提示（Modeling）**: 読み上げ音声でお手本を先に聞ける（Web Speech API / 速度調整可）
- **正確さ＋理解を併記**: 「すらすら/つっかえた」自己申告と、各文の内容確認（2択）を記録
- **速度は強制しない**: 自動送り（強制ペース）は廃止。「きのうの じぶん」比の自己ベスト改善を可視化
- **分散復習（Spacing）**: 数日あいた短文を優先的に再出題（`💫 ふくしゅう`表示）
- **ゲーミフィケーション**: スコア・コンボ・連続日数（streak）・スター/バッジ（報酬は正確さ・理解に重みづけ）
- 約1分セッション
- **問題のバリエーション**：1年（ひらがな／カタカナ入り／かんじ入り）・2年（ひらがな／かんじ入り）・**3〜6年（かんじ入り）**。漢字込み問題は[学年別漢字配当表](docs/kanji-by-grade.md)に沿った漢字のみを使用（1・2年は各学年の全漢字を網羅、3〜6年は累積漢字の範囲で出題）
- **大人むけ（こどもと いっしょに）**：小学校で習わない常用漢字を多く含む文を30問収録。親子で交代に読み合うのに最適
- 成績のローカル保存（`localStorage`）

> 反復読み（流暢性）型の本手法は文献上**小6まで有効**（効果量は学年が上がるほど逓減）。**中学生以降は手法が変わる**ため、別アプリ「速読・読解トレーニング」を用意しています。

## 速読・読解トレーニング（中高生むけ）の機能

中学生以降に有効な「目的に応じた読みの使い分け」を訓練します（[secondary/index.html](secondary/index.html)）。

- **3つの読み方モード**：ざっくり読み（スキミング＝要旨把握）／さがし読み（スキャニング＝情報探索）／じっくり読み（精読＝細部＋推論）
- **読解方略の明示プロンプト**：各モードの前にコツ（見出し活用・要約・自己質問・モニタリング等）を提示
- **理解の担保**：読速（文字/分）と理解（設問の正誤）を併記し、**理解できた時だけ「有効読速」を記録**（速度‑正確性トレードオフへの配慮）
- モード別ベスト有効読速・通算理解率をローカル保存
- RSVP型の高速強制提示は**採用しない**（詳細理解を上げないため）

> 速度と理解の両立をうたう「速読術」やRSVP型の高速強制提示は、低学年では理解低下のおそれがあるため採用していません。詳細は調査ドキュメントを参照してください。

## ファイル構成

小学生むけ（よむよむスピードトレーニング）

- `index.html` / `styles.css` / `app.js` … 画面・スタイル・ロジック
- `data/passages-grade1.js` … 1年生の問題（ひらがな／カタカナ入り／かんじ入り）
- `data/passages-grade2.js` … 2年生の問題（ひらがな／かんじ入り）
- `data/passages-grade3to6.js` … 3〜6年生の問題（かんじ入り）
- `data/passages-adult.js` … 大人むけの問題
- `app.standalone.html` … 全部を1ファイルにまとめた版（Artifact用／単体配布用）

問題データは各 `data/passages-*.js` が `window.PASSAGE_SETS` に登録し、`app.js` がそれを読み込みます（`index.html` で `app.js` より前に読み込み）。問題を増やすときは該当する `data/` ファイルを編集してください。

中高生むけ（速読・読解トレーニング）

- `secondary/index.html` / `secondary/styles.css` / `secondary/app.js` … 画面・スタイル・ロジック
- `secondary/passages.js` … 文章＋3種の設問（要旨／探索／精読）
- `secondary/app.standalone.html` … 1ファイル版

共通

- `tools/build-standalone.py` … 2つの `app.standalone.html` を再生成するスクリプト

単一ファイル版を更新するには：

```bash
python3 tools/build-standalone.py
```

## デプロイ

このリポジトリは静的ファイルのみのため、`index.html`, `styles.css`, `app.js`, `data/`, `secondary/`, `docs/` をそのまま配置できる環境ならデプロイ可能です。単一ファイルで済ませたい場合は各 `app.standalone.html` だけでも動きます。
